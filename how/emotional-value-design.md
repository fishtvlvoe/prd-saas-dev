# 情緒價值設計法（Emotional Value Design）

這個檔案解決的問題：當「功能滿足」已經不夠賣的時候，怎麼用神經遞質視角設計產品，讓使用者**情緒上**買單，而不只是**功能上**買單。適用於補習班、健身、保險、保健品、心理諮商等所有「販售焦慮解藥」的產業。

---

## 核心命題

人不是為了功能付錢，是為了**情緒狀態改變**付錢。

```
表層需求：我要孩子英文進步
深層需求：我要不再焦慮、要感覺自己是好爸媽
   ↑
真正會掏錢買單的層
```

賣表層 = 紅海競爭（每家都在賣英文進步）
賣深層 = 藍海定位（解情緒痛苦的補習班只有一家）

---

## 四種神經遞質 → 四種設計切角

| 神經遞質 | 對應情緒 | 觸發場景 | 設計手法 |
|---------|---------|---------|---------|
| **皮質醇 ↓** | 焦慮降低 | 用戶覺得「我選對了」「我有掌控」 | 確定感、即時止痛、案例佐證 |
| **多巴胺 ↑** | 期待 / 小勝利 | 看到下一步、看到進展 | 進度可視化、每日小驚喜、預告 |
| **血清素 ↑** | 被認同 / 地位 | 被肯定、感覺自己不一樣 | 社會認同、勳章、「你是好 X」 |
| **催產素 ↑** | 連結 / 歸屬 | 同類社群、被理解 | 私域、戰友配對、共同經歷 |

完整方案要四個都打到，但 MVP 階段優先打**皮質醇 ↓**（止痛），其他作為強化。

---

## 五步驟設計流程

### Step 1：找出真實的「焦慮觸發鏈」

不要從「用戶需要 X 功能」開始，從「用戶在焦慮什麼」開始。

格式：
```
觸發事件 → 認知投射 → 比較放大 → 生理反應 → 行為輸出
```

範例（家長焦慮孩子英文）：
```
孩子英文不好 → 投射「考不到好學校」→ 跟別家小孩比 → 皮質醇↑ 失眠 → 找補習班 / 滑社群焦慮比較
```

### Step 2：辨識「爽點 vs 痛點 vs 摩擦力」

| 維度 | 爽點 | 痛點 | 摩擦力 |
|------|------|------|--------|
| 用了會 | 開心、轉發 | 不焦慮、能睡 | 增加負擔、想退 |
| 不用會 | 沒事 | 持續焦慮 → 找競品 | 解脫 |
| 範例 | 成果影片 | AI 即時情緒陪伴 | 每晚親子作業 |

設計守則：**用痛點當主菜，爽點當配菜，摩擦力一律砍掉**。

### Step 3：對應神經遞質設計處方箋

把痛點按「降皮質醇 → 加多巴胺 → 加血清素 → 加催產素」順序開處方：

```
止痛（必要） → 期待（必要） → 認同（強化）→ 歸屬（黏著）
```

每個處方對應具體產品功能 / 內容 / 互動點。

### Step 4：核對五條紅線

開始實作前，每個設計點過一次紅線檢查：

- [ ] **真實性**：神經遞質只在「真實發生的好事」上觸發，不造假
- [ ] **底層空心檢查**：情緒服務必須建立在真實價值之上（補習班教學品質、保健品有效成分等）
- [ ] **上癮 vs 安心離開**：用戶能「感到夠了 = 停」，不是「離不開 = 賺」
- [ ] **私域不變焦慮放大器**：不公開排名、不展示絕對水平、過濾炫耀內容
- [ ] **資料隱私**：私訊內容不訓練 AI、不跨用戶分享

任一條碰到 → 重新設計，不存在「特殊例外」。

### Step 5：MVP 篩選 — 從處方箋裡挑最小痛點殺手

選擇判準：
1. 對用戶負擔 = 零（被動接收 + 主動點才用）
2. 即時止痛效果最強（皮質醇下降最快）
3. 既有資產可改造（不需重起爐灶）

## 反模式（不要這樣做）

- ❌ **賣表層功能不賣深層情緒**：每家都這樣 → 紅海死戰
- ❌ **設計上癮機制**：IG/TikTok 模式 → 短期黏著、長期反噬（用戶會恨你）
- ❌ **造假觸發神經遞質**：編孩子進步、編家長見證 → 詐騙、毀牌
- ❌ **情緒服務建立在空心底層**：賣解藥但商品本身沒效 → 麻醉劑 → 長期崩盤
- ❌ **要求用戶配合工具行為**：每晚親子作業、每天打卡 → 摩擦力炸裂
- ❌ **私域內容變炫耀牆**：別人小孩多優秀 → 焦慮放大器 → 自殺

---

## 商業競爭力檢查

用情緒價值設計的產品，要有以下競爭力才值得做：

| 維度 | 為什麼 |
|------|-------|
| 競品沒做 | 大家都在賣功能，沒人賣情緒 = 藍海 |
| 黏著度高 | 私域 + 情緒依附 = 高轉換成本 |
| 自帶傳播 | 同類焦慮者會拉同類進來 |
| 模板可複製 | 焦慮場景跨產業可遷移 |
| 持續價值 | 情緒需求週期長（補習 5-10 年、健身終身、保健終身） |
| AI 邊際成本低 | 一套內容引擎服務萬人 |

少於 4 條 → 不值得做這條路。
